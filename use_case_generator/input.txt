 
Cas d'utilisation : Gestion des commandes 
toujours ke nom.

Acteurs: Client, Système de commande

Préconditions: Le client doit être inscrit sur la plateforme.
PostConditions: La commande est enregistrée avec succès.

Scénario principal:
Le client se connecte à son compte.
Le client ajoute un article au panier.
Le client confirme la commande.

Scénario alternatif:
Le client ajoute un article au panier sans se connecter.
Le système demande une connexion avant la confirmation.

Cas d'utilisation : Génération de rapports.

Acteurs: Analyste, Système de reporting

Préconditions: Les données doivent être disponibles dans le système.
PostConditions: Le rapport est généré avec succès.

Scénario principal:
L'analyste sélectionne les données.
Le système génère un rapport PDF.

Scénario alternatif:
L'analyste charge manuellement les données.
Le rapport est généré partiellement.

Cas d'utilisation : Suivi des livraisons.

Acteurs: Client, Livreurs, Système de suivi

Préconditions: La commande doit être expédiée.
PostConditions: Le client voit l'état de livraison.

Scénario principal:
Le client consulte l'état de la commande.
Le système affiche le statut actuel.

Scénario alternatif:
Le système indique un retard de livraison.
Une notification est envoyée au client.

Cas d'utilisation : Gestion des utilisateurs.

Acteurs: Administrateur, Utilisateur

Préconditions: L'administrateur doit être connecté.
PostConditions: Les informations utilisateur sont mises à jour.

Scénario principal:
L'administrateur sélectionne un utilisateur.
L'administrateur modifie les informations.
Le système enregistre les modifications.

Scénario alternatif:
Les informations fournies sont incomplètes.
Le système affiche un message d'erreur.

Cas d'utilisation : Notification des événements.

Acteurs: Utilisateur, Système de notification

Préconditions: L'utilisateur est inscrit aux notifications.
PostConditions: Une notification est reçue avec succès.

Scénario principal:
Un événement important est déclenché.
Le système envoie une notification à l'utilisateur.

Scénario alternatif:
L'utilisateur désactive les notifications.
Le système enregistre l'option de l'utilisateur .
